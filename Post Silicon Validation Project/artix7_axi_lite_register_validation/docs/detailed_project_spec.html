Subsystem Architecture Specifications

The register validation framework consists of 7 tightly integrated subsystems designed to bridge physical serial communication with the internal AXI-Lite bus lines. Below is an in-depth breakdown of each subsystem:

Subsystem A: UART Transceivers

uart_rx & uart_tx

The UART receiver (uart_rx) and transmitter (uart_tx) handle raw serial-to-parallel and parallel-to-serial conversion. The receiver uses a 16x oversampling clock generator to identify the start bit. It samples the serial line in the middle of each bit time-slot to maximize noise immunity and eliminate transmission clock jitter. The oversampling counter acts as a digital filter, checking that the RX line remains stable before latching data bits into the shift register. Once a complete 8-bit frame is assembled and the stop bit is validated, the module asserts a single-cycle rx_valid pulse.

Subsystem B: UART Register Interface

uart_register_interface

This module acts as the byte-to-command deserializer and assembler. It implements an internal state machine (comprising states like ST_WAIT_OPCODE, ST_ADDR_X, ST_DATA_X, and ST_SEND_RESP) to collect incoming bytes. A write packet requires 9 sequential bytes (Opcode + 4 Address Bytes + 4 Data Bytes) while a read packet requires 5 bytes. The module coordinates the transmission of status bytes and read values back through the UART transmitter after the AXI-Lite master completes a transaction, handling the response serialization sequence byte-by-byte.

Subsystem C: AXI-Lite Master Command Interface

axi_lite_master_cmd_if

The AXI Master is the active driver of the AXI-Lite channels (AW, W, B, AR, R). It translates command signals from the UART interface into formal bus handshake cycles, asserting AWVALID and WVALID for write cycles or ARVALID for read cycles. To prevent the bus from locking up if a slave fails to respond, it contains a 16-bit watchdog timeout counter. If a handshake takes more than 1024 clock cycles to complete, the master cancels the transaction, asserts timeout_error, and returns a SLVERR response status back to the controller.

Subsystem D: AXI-Lite Error Injector

axi_lite_error_injector

Positioned inline on the bus, the error injector acts as a debugging module. It routes AXI-Lite channel signals through multiplexers controlled by debug lines. During verification testing, it can inject artificial handshaking stalls (for example, by holding the READY lines low) to test the master's timeout counter, or corrupt read data bits to verify that the python validation suite flags data integrity faults correctly.

Subsystem E: AXI-Lite Interconnect

axi_lite_interconnect

The interconnect acts as the address decoder and bus router. It decodes bits [31:8] of the address bus to select exactly one of the three register blocks:

  • GPIO block selected if address is in range 0x00000000 - 0x000000FF.
  • Status block selected if address is in range 0x00000100 - 0x000001FF.
  • Timer block selected if address is in range 0x00000200 - 0x000002FF.
If the address does not match any block, the interconnect routes the transaction to a dummy decoder that immediately returns a Decode Error (DECERR) response.

Subsystem F: Register Blocks

gpio, status, & timer regs

The register blocks house the actual memory cells. Each register block is built on top of a reusable, parameterized module (axi_lite_register_block) that supports configurable write masks and default reset values. It handles byte write strobes (wstrb) to allow writing to individual bytes of a 32-bit register, and enforces the programmed access permissions (RW, RO, WO) to ensure read-only status registers or write-only command registers behave as specified.

Subsystem G: Passive Monitors

protocol, reset, & access monitors

These modules passively observe the AXI-Lite bus to provide real-time hardware status metrics. The Protocol Checker checks that address and data lines remain stable while VALID is high but before READY handshakes, latching any violations. The Access Monitor counts successful read/write handshakes and logs the last accessed address offset. The Reset Monitor tracks the post-reset stabilization timer window, flagging any premature master transactions that occur before the reset recovery period completes.

clk INPUT wire System Clock (50 MHz)

mon_awaddr INPUT wire [31:0] Observed write address lines under monitoring
mon_awvalid INPUT wire Observed write address valid line under monitoring
mon_awready INPUT wire Observed write address ready line under monitoring
mon_araddr INPUT wire [31:0] Observed read address lines under monitoring
mon_arvalid INPUT wire Observed read address valid line under monitoring
mon_arready INPUT wire Observed read address ready line under monitoring
protocol_violation OUTPUT wire Asserts high and locks if any AXI protocol violation is detected
access_count OUTPUT wire [31:0] Running register counting successful read/write bus operations
last_addr OUTPUT wire [31:0] Latch containing the address of the most recent access operation
reset_stable OUTPUT wire Status line indicating active reset stabilization phase is complete

Active-Low Logic (rst_n)

Connected to the center push button. Active-high physically, inverted to active-low in RTL, with a 16-cycle stabilization window.

AXI-Lite Status Response Codes

The status byte (Byte 0 of Response packets) encapsulates the low-level AXI-Lite bus channel errors mapped to standard UART response fields:

Status Code Name AXI Channel Equivalent Operational Meaning
0x00 OKAY BRESP / RRESP = 2'b00 Transaction completed successfully. Slave acknowledged write address/data or read request.
0x01 EXOKAY BRESP / RRESP = 2'b01 Exclusive access okay (reserved). Indicates non-standard bus status warning.
0x02 DECERR BRESP / RRESP = 2'b11 Decode Error. The decoded address maps outside of GPIO, Status, or Timer ranges.
0x03 TIMEOUT Internal Watchdog Latch The target slave failed to assert READY inside the 16-cycle transaction window. Master aborted.
